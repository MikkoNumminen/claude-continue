# Robustness audit — 2026-06-22

**Scope:** `src/claude_continue/` (v0.10.0, 22 modules) + tests
**Detected shape:** Python, stdlib-only CLI + Tkinter GUI; security surface = GitHub
self-update (download / SHA-256 verify / binary swap), `subprocess`/`cmd`/`powershell`,
Windows registry, Task Scheduler, console injection.
**Method:** `mikko-audit` methodology — six parallel scoped finders (resource lifecycle,
data integrity, concurrency, error paths, external boundaries/injection, AI-codegen
smells), **each finding then adversarially verified** against the real code before being
counted. Read-only.

## Rejected (verified NOT defects on `main`)

- `cleanup_stale_update` reaps every `cc-update-*` temp dir unconditionally — a
  pre-existing, bounded, non-bricking race carried forward, not introduced here.
- `release.yml` `Compress-Archive` forward-slash dependency — not a realizable break
  (the release runs under `shell: pwsh`, which emits `/` entries; native Windows
  handles both).
- A couple of test-coverage observations on already-correct code (speculative).

## Note on the review loop

Each remediation PR was adversarially reviewed before merge, and the review caught a
**real defect in every first cut**: PR #40 (an embedded `"` re-exposed cmd operators —
verified against real `cmd.exe`), PR #41 (`OverflowError` from an extreme timestamp
still escaped the widened `except`), PR #42 (a JSON `null` release body still hit the
bare `assert`). All were fixed, re-verified, and merged green. The fixes ship under
`[Unreleased]` in `CHANGELOG.md`.
